# AmengCloud 项目详细说明文档

## 一、项目概述

### 1.1 项目定位
AmengCloud 是一个基于 Electron 框架开发的跨平台桌面云存储应用程序，专门用于管理 workdrive 挂载盘的文件存储与下载。项目采用前后端分离架构，前端负责用户界面展示与交互，后端（主进程）负责文件系统操作、网络请求和数据处理。

### 1.2 技术栈详情
| 技术领域 | 技术选型 | 版本/说明 |
|---------|---------|---------|
| 桌面框架 | Electron | v41.2.1 |
| 构建工具 | Electron Forge | v7.11.1 |
| 前端技术 | HTML5, CSS3, JavaScript | ES6+ |
| 图标库 | Font Awesome | 免费版 |
| 数据库 | BaseMulti 低代码平台 | 云端 NoSQL |
| 开发环境 | Node.js | v16+ |
| 包管理工具 | npm | 随 Node.js 安装 |
| 加密算法 | SHA-256 | 文件完整性校验 |
| 压缩算法 | Gzip | 数据压缩存储 |

### 1.3 项目核心特性
- ✅ 用户登录/注册与身份认证
- ✅ 文件上传（支持大文件分片存储）
- ✅ 文件下载（支持分片合并）
- ✅ 文件删除（级联删除分片数据）
- ✅ 文件夹层级管理
- ✅ 传输队列（FIFO 顺序下载）
- ✅ 实时进度显示
- ✅ 深色/浅色主题切换
- ✅ 响应式界面设计
- ✅ 数据完整性校验（SHA256）
- ✅ 数据压缩存储（Gzip）
- ✅ 文件分享功能（支持密码保护）

## 四、进程通信（IPC）详解

### 4.1 IPC 架构原则
- ✅ **上下文隔离 (Context Isolation)**: 渲染进程无法直接访问 Node.js API
- ✅ **预加载脚本 (Preload Script)**: 仅暴露必要的安全 API 给渲染进程
- ✅ **单向数据流**: 渲染进程通过 invoke 调用主进程 API，主进程通过 send 发送事件
- ✅ **事件清理**: 注册新事件监听器前先移除旧监听器，防止内存泄漏

### 4.2 预加载脚本 (preload.js) API 详解

#### 4.2.1 文件选择与读取 API
```javascript
electronAPI.selectFile()
// → 打开文件选择对话框
// 返回: 选中的文件路径 (String) 或 null (取消)

electronAPI.readFileAsBase64(filePath)
// → 读取文件并转换为 Base64
// 参数: filePath - 文件路径 (String)
// 返回: Base64 编码的文件内容 (String)
```

#### 4.2.2 文件上传 API
```javascript
electronAPI.calculateSHA256(filePath)
// → 计算文件的 SHA256 哈希值
// 参数: filePath - 文件路径 (String)
// 返回: SHA256 哈希字符串 (String)

electronAPI.getTotalChunks(filePath)
// → 计算文件需要的分片总数
// 参数: filePath - 文件路径 (String)
// 返回: { success: true, totalChunks: Number }

electronAPI.uploadLargeFile(params, totalChunks, updateProgress)
// → 分片上传大文件
// 参数:
//   params - { filePath, fileName, sha256, floder }
//   totalChunks - 分片总数 (Number)
//   updateProgress - 进度回调函数 (Function)
//     → 参数: current, total, message
// 返回: 上传结果对象

electronAPI.uploadFile(fileData)
// → 上传小文件（旧格式，不推荐使用）
// 参数: fileData - { name, base64, sha256, floder }
// 返回: 上传结果对象
```

## 五、传输队列系统详解

### 5.1 传输队列架构图
```
┌─────────────────────────────────────────────────────────────┐
│                    用户点击下载文件                         │
└────────────────────────────┬────────────────────────────────┘
                             │
                             ▼
┌─────────────────────────────────────────────────────────────┐
│              addToDownloadQueue(fileName)                  │
│  1. 检查文件是否已在队列中（防止重复添加）                  │
│  2. 获取默认下载目录 (localStorage)                        │
│  3. 构建任务对象 { id, fileName, savePath, status, ... }  │
│  4. 添加到 downloadQueue 数组                             │
│  5. 调用 updateQueueUI() 更新界面                          │
│  6. 调用 startDownloadQueue() 启动队列处理                 │
└────────────────────────────┬────────────────────────────────┘
                             │
                             ▼
┌─────────────────────────────────────────────────────────────┐
│                    startDownloadQueue()                    │
│  1. 检查 isDownloading 标志（防止并发下载）                │
│  2. 查找第一个 status === 'pending' 的任务                 │
│  3. 设置 isDownloading = true                              │
│  4. 设置任务状态为 'downloading'                           │
│  5. 调用 electronAPI.downloadFile() 开始下载               │
└────────────────────────────┬────────────────────────────────┘
                             │
                             ▼
┌─────────────────────────────────────────────────────────────┐
│                  下载进行中...                             │
│  → 进度更新触发 updateTaskProgress()                       │
│  → 更新任务的 progress, progressText, chunkInfo           │
│  → 调用 updateQueueUI() 刷新界面                          │
└────────────────────────────┬────────────────────────────────┘
                             │
              ┌──────────────┴──────────────┐
              │                             │
              ▼ (成功)                      ▼ (失败)
    ┌─────────────────┐            ┌─────────────────┐
    │  下载完成处理   │            │  下载失败处理   │
    └────────┬────────┘            └────────┬────────┘
             │                              │
             ▼                              ▼
┌───────────────────────────┐  ┌───────────────────────────┐
│ 任务状态 → 'completed'   │  │ 任务状态 → 'error'       │
│ progress → 100           │  │ progressText → '下载失败' │
│ 显示成功通知             │  │ 显示失败通知             │
└────────────┬──────────────┘  └────────────┬──────────────┘
             │                              │
             └──────────────┬───────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────┐
│  1. 设置 isDownloading = false                             │
│  2. 清除 currentDownloadTask                              │
│  3. 调用 updateQueueUI()                                  │
│  4. 延迟 500ms 后再次调用 startDownloadQueue()            │
│     → 自动开始下一个任务（FIFO）                           │
└─────────────────────────────────────────────────────────────┘
```

### 5.2 传输队列任务对象结构
```javascript
{
  id: 1680000000000,        // 任务 ID（时间戳）
  fileName: "document.pdf",  // 文件名
  savePath: "C:/Downloads/document.pdf",  // 保存路径
  status: "pending",         // 状态: pending | downloading | completed | error
  progress: 0,               // 进度百分比 (0-100)
  progressText: "等待下载",   // 进度文本
  chunkInfo: ""              // 分片信息（例如: "第 3/5 个分片"）
}
```

### 5.4 传输队列核心函数详解

#### 5.4.1 initTransferQueue() - 初始化函数
```javascript
功能:
  1. 绑定侧边栏导航点击事件
  2. 绑定清空队列按钮点击事件
  3. 修改右键菜单下载按钮行为（改为添加到队列）

调用时机:
  DOMContentLoaded 事件后，应用初始化时调用
```

#### 5.4.2 showView(view) - 视图切换函数
```javascript
参数:
  view - "files" | "queue"

功能:
  1. 根据 view 参数显示/隐藏对应视图
  2. 更新导航项的 active 状态
  3. 更新顶部标题栏文本
  4. 显示/隐藏路径导航区域
```

#### 5.4.3 addToDownloadQueue(fileName) - 添加任务函数
```javascript
参数:
  fileName - 文件名（String）

功能:
  1. 检查文件是否已在队列中（通过 fileName 去重）
  2. 从 localStorage 获取默认下载目录
  3. 构建完整保存路径
  4. 创建任务对象（ID 使用时间戳）
  5. 添加到 downloadQueue 数组
  6. 更新 UI 并启动队列
```

#### 5.4.4 startDownloadQueue() - 启动队列函数
```javascript
功能:
  1. 检查 isDownloading 标志（防止并发）
  2. 查找第一个 pending 任务
  3. 标记为 downloading 状态
  4. 调用 electronAPI.downloadFile()
  5. 监听下载进度并更新 UI
  6. 完成/失败后重置状态并启动下一个任务

注意:
  - 使用 setTimeout 延迟 500ms 启动下一个任务
  - 这给了 UI 更新和用户交互的时间窗口
```

#### 5.4.5 updateTaskProgress() - 更新进度函数
```javascript
参数:
  taskIndex - 任务索引
  current - 当前进度值
  total - 总进度值
  message - 状态消息

功能:
  1. 计算百分比: Math.round((current/total)*100)
  2. 从 message 中提取分片信息（正则匹配）
  3. 更新任务对象的 progress, progressText, chunkInfo
  4. 调用 updateQueueUI() 刷新界面
```

#### 5.4.6 updateQueueUI() - 更新界面函数
```javascript
功能:
  1. 更新队列徽章数量（pending + downloading）
  2. 如果队列为空，显示空队列提示
  3. 否则，渲染所有任务卡片
  4. 根据任务状态渲染不同的 UI 元素

渲染模板:
  使用 Array.map() + 模板字符串动态生成 HTML
  使用三元运算符条件渲染元素
```

#### 5.4.7 removeFromQueue(index) - 删除任务函数
```javascript
参数:
  index - 任务索引

功能:
  1. 从 downloadQueue 数组中移除指定索引的任务
  2. 调用 updateQueueUI() 刷新界面
```

#### 5.4.8 stopDownload(index) - 停止下载函数
```javascript
参数:
  index - 任务索引

功能:
  1. 将任务状态从 'downloading' 改回 'pending'
  2. 重置进度和状态文本
  3. 设置 isDownloading = false
  4. 清空 currentDownloadTask
  5. 更新 UI 并重启队列
```

#### 5.4.9 retryDownload(index) - 重试下载函数
```javascript
参数:
  index - 任务索引

功能:
  1. 将任务状态从 'error' 改为 'pending'
  2. 重置进度和状态文本
  3. 更新 UI 并重启队列
```

#### 5.4.10 clearDownloadQueue() - 清空队列函数
```javascript
功能:
  1. 清空 downloadQueue 数组
  2. 重置 isDownloading 标志
  3. 清空 currentDownloadTask
  4. 更新 UI
```

## 八、常见问题与解决方案

### 8.1 文件上传问题

#### Q: 上传大文件时浏览器卡住怎么办？
A: ✅ 已使用主进程处理分片上传，不会阻塞渲染进程

#### Q: 如何知道上传进度？
A: ✅ 通过 upload-progress 事件实时更新进度

### 8.2 文件下载问题

#### Q: 下载中断后可以续传吗？
A: ❌ 当前版本不支持断点续传，需要重新下载

#### Q: 如何校验下载的文件是否完整？
A: ✅ Files 表存储了 SHA256 哈希值，可以校验（当前版本未自动校验，可手动实现）

### 8.3 传输队列问题

#### Q: 关闭应用后队列会保存吗？
A: ❌ 当前版本不会保存队列状态，重启后队列清空

#### Q: 可以调整下载并发数吗？
A: ❌ 当前版本固定为 FIFO 单线程下载，不支持并发
