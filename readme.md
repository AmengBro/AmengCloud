# AmengCloud 项目详细说明文档

## 一、项目概述

### 1.1 项目定位
AmengCloud 是一个基于 Electron 框架开发的跨平台桌面云存储应用程序，专门用于管理垃圾后端数据库文件存储与下载。项目采用前后端分离架构，前端负责用户界面展示与交互，后端（主进程）负责文件系统操作、网络请求和数据处理。

### 1.2 技术栈详情
| 技术领域 | 技术选型 | 版本/说明 |
|---------|---------|---------|
| 桌面框架 | Electron | v41.2.1 |
| 构建工具 | Electron Forge | v7.11.1 |
| 前端技术 | HTML5, CSS3, JavaScript | ES6+ |
| 图标库 | Font Awesome | 免费版 |
| 数据库 | BaseMulti 低代码平台 | 云端 NoSQL |
| 开发环境 | Node.js | v16+ |
| 包管理工具 | npm | 随 Node.js 安装 |
| 加密算法 | SHA-256 | 文件完整性校验 |
| 压缩算法 | Gzip | 数据压缩存储 |

### 1.3 项目核心特性
- ✅ 用户登录/注册与身份认证
- ✅ 文件上传（支持大文件分片存储）
- ✅ 文件下载（支持分片合并）
- ✅ 文件删除（智能判断：单用户删除源文件，多用户仅取消共享）
- ✅ 文件夹层级管理
- ✅ 传输队列（FIFO 顺序下载）
- ✅ 实时进度显示
- ✅ 深色/浅色主题切换
- ✅ 响应式界面设计
- ✅ 数据完整性校验（SHA256）
- ✅ 数据压缩存储（Gzip）
- ✅ 文件分享功能（支持密码保护）

## 四、文件分享功能详解

### 4.1 分享链接格式
分享链接采用伪链接格式，支持密码保护：
```
amengshare://{Id}.sharedameng.{随机数}.file
```
- `Id`: 分享记录在数据库中的 ID
- `随机数`: 用于让链接看起来更自然的随机字符（解析时被忽略）

- 如果要分享的文件不在根目录（floder != 0），系统会提示："你也许需要分享其父目录才会给对方展示该文件"
- 接收文件时，被分享文件的 ID 会被添加到接收方的 owned_file 中
- 密码留空表示无密码保护

## 五、删除逻辑详解

### 5.1 智能删除判断
当用户删除文件时，系统会检查该文件是否被其他用户引用：

| 场景 | 条件 | 操作 |
|------|------|------|
| 无人引用 | 无其他用户的 owned_file 包含此文件 | 删除源文件 + 从当前用户 owned_file 移除 |
| 只有自己引用 | 只有当前用户的 owned_file 包含此文件 | 删除源文件 + 从当前用户 owned_file 移除 |
| 有他人引用 | 有其他用户的 owned_file 也包含此文件 | 只从当前用户 owned_file 移除，显示"已取消共享权限" |

### 5.2 文件夹删除
删除文件夹时会级联删除所有子文件：
1. 先删除所有子文件（每个子文件按上述逻辑判断）
2. 再删除文件夹本身

## 七、开发规范

### 7.1 代码风格规范

#### 7.1.1 命名约定
| 元素类型 | 命名规则 | 示例 |
|---------|---------|------|
| 文件 | 小写，下划线分隔 | `index.js`, `user_profile.html` |
| 变量/函数 | 小驼峰 (camelCase) | `currentOperateId`, `fetchFiles()` |
| 常量 | 大写下划线分隔 (UPPER_SNAKE_CASE) | `CHUNK_SIZE`, `API_KEY` |
| CSS 类名 | 小写连字符分隔 (kebab-case) | `file-item`, `queue-view` |
| CSS ID | 小驼峰或连字符 | `headerTitle`, `queue-badge` |

#### 7.1.2 注释规范
```javascript
// 单行注释：简短说明
let count = 0;

/**
 * 多行函数注释（JSDoc 风格）
 * @param {string} fileName - 文件名
 * @param {string} savePath - 保存路径
 * @returns {Promise<object>} 下载结果
 * @throws {Error} 下载失败时抛出错误
 */
async function downloadFile(fileName, savePath) {
  // ...
}

/*
 * 代码块注释
 * 用于说明复杂逻辑或重要业务规则
 */
```

#### 7.1.3 代码组织原则
- ✅ **单一职责**: 每个函数只做一件事
- ✅ **DRY 原则**: 避免重复代码，抽取公共函数
- ✅ **按功能分组**: 相关函数放在一起
- ✅ **自顶向下**: 主函数在前，辅助函数在后

### 7.2 Git 提交规范

#### 7.2.1 提交信息格式
```
<type>(<scope>): <subject>

<body>

<footer>
```

#### 7.2.2 Type 类型说明
| Type | 说明 | 示例 |
|------|------|------|
| feat | 新功能 | `feat(queue): 添加传输队列功能` |
| fix | 修复 bug | `fix(upload): 修复分片上传失败问题` |
| docs | 文档更新 | `docs: 更新 what_we_do.md 文档` |
| style | 代码格式调整 | `style: 格式化 CSS 文件` |
| refactor | 重构（不改变功能） | `refactor: 重构文件上传逻辑` |
| test | 测试相关 | `test: 添加文件下载测试` |
| chore | 构建/工具相关 | `chore: 更新依赖包版本` |

#### 7.2.3 Scope 说明
Scope 表示影响的模块，例如:
- `queue` - 传输队列模块
- `upload` - 文件上传模块
- `download` - 文件下载模块
- `auth` - 用户认证模块
- `ui` - 界面相关
- `api` - API 相关

#### 7.2.4 提交示例
```
feat(queue): 实现 FIFO 传输队列系统

- 添加队列管理核心函数
- 实现下载进度显示
- 添加任务控制按钮（删除/停止/重试）
- 实现队列徽章提示

Closes #123
```

## 八、常见问题与解决方案

### 8.1 文件上传问题

#### Q: 上传大文件时浏览器卡住怎么办？
A: ✅ 已使用主进程处理分片上传，不会阻塞渲染进程

#### Q: 如何知道上传进度？
A: ✅ 通过 upload-progress 事件实时更新进度

### 8.2 文件下载问题

#### Q: 下载中断后可以续传吗？
A: ❌ 当前版本不支持断点续传，需要重新下载

#### Q: 如何校验下载的文件是否完整？
A: ✅ Files 表存储了 SHA256 哈希值，可以校验（当前版本未自动校验，可手动实现）

### 8.3 传输队列问题

#### Q: 关闭应用后队列会保存吗？
A: ❌ 当前版本不会保存队列状态，重启后队列清空

#### Q: 可以调整下载并发数吗？
A: ❌ 当前版本固定为 FIFO 单线程下载，不支持并发

### 8.4 文件删除问题

#### Q: 删除文件会同时删除源文件吗？
A: ✅ 智能判断：如果只有你一个人拥有该文件的访问权限，则删除源文件；否则只取消你的共享权限

#### Q: 如何彻底删除一个被多人引用的文件？
A: ❌ 当前版本不支持强制删除多人引用的文件，需要联系其他用户先取消共享
